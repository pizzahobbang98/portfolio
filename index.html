Selected Projects

대표 프로젝트

많은 프로젝트를 나열하기보다, 직접 설명할 수 있고 문제 정의부터 구현 흐름까지 말할 수 있는 프로젝트만 정리했습니다.

Problem

임산부는 컨디션 변화가 잦지만, 매번 앱에 상태를 입력하거나 여러 가전을 하나씩 조작하기 어렵습니다.

Built

음성 발화, AI 의도 분석, ThinQ 공기청정기 제어, Hue 조명 제어, 3D 홈 시뮬레이터 반응, Supabase 기록 저장을 연결했습니다.

Role

서비스 기획, 데이터 분석, 프론트엔드 구현, API 연동, 시연 환경 구성까지 프로젝트 전반을 담당했습니다.

Next.js Vercel Supabase OpenAI API LG ThinQ API FastAPI Python 3D Simulator Philips Hue
핵심 포인트. 단순 화면 프로토타입이 아니라, 사용자의 음성 입력부터 AI 판단, 가전 실행, 조명 반응, 3D 공간 변화, 실행 로그 저장까지 이어지는 시연 가능한 MVP로 구현했습니다.

Problem

스탠바이미 Go처럼 고객이 새로운 사용 상황을 떠올리게 하는 제품 기회를 또 발견할 수 있을까? 또한 LGE.COM에서 고객이 탐색·구매 과정 중 어떤 지점에서 불편을 느끼는지 데이터 기반으로 개선할 수 있을까?

Built

커뮤니티와 리뷰 데이터를 수집·전처리하고, 고객이 실제로 표현한 상황, 불편, 니즈를 분류해 제품 기회와 UX 개선 포인트로 구조화했습니다.

Result

BX 프로젝트에서는 여행 중 의류 관리 니즈를 바탕으로 AI Styler Go 컨셉을 도출했고, CX 프로젝트에서는 고객이 제품 정보를 찾고 비교한 뒤 구매를 결정하는 과정에서 겪는 불편을 분석해 LGE.COM의 탐색 흐름, 정보 배치, 구매 전환 UX 개선 방향을 제안했습니다.

Python Pandas Selenium Web Crawling Text Preprocessing TF-IDF K-means LDA Wordcloud
핵심 포인트. 단순히 데이터를 수집하는 데서 끝내지 않고, 실제 사용자의 언어를 기반으로 니즈와 페인포인트를 정리한 뒤 제품 컨셉과 UX 개선 방향으로 연결했습니다.
Project Focus

제가 보여드리고 싶은 강점

“특정 기술을 많이 썼다”보다, 문제를 이해하고 실제 동작하는 흐름으로 연결한 경험을 중심으로 작성했습니다.

01

문제 정의

기술을 먼저 정하지 않고, 사용자가 어떤 상황에서 불편을 겪는지부터 정의합니다.

02

데이터 기반 사고

사용자 발화, 리뷰, 커뮤니티 데이터를 분석해 문제를 구조화하는 경험을 쌓았습니다.

03

API 연동

AI 판단 결과를 외부 API와 연결해 실제 화면과 기기가 반응하도록 구현했습니다.

04

서비스 구현

분석 결과를 문서에서 끝내지 않고, 사용자가 체감할 수 있는 서비스 흐름으로 연결합니다.
