Selected Projects

대표 프로젝트

많은 프로젝트를 나열하기보다, 직접 설명할 수 있고 문제 정의부터 구현 흐름까지 말할 수 있는 프로젝트만 정리했습니다.

Problem

임산부는 신체와 감정 상태가 자주 변하지만, 매번 앱에 상태를 기록하거나 여러 가전을 직접 조작하기 어렵습니다. 배우자 또한 언제 어떤 도움이 필요한지 바로 알기 어렵습니다.

Approach

임신 관련 VOC를 수집·전처리하고, 임베딩·계층적 군집·토픽 분석·중요도와 만족도 기반 기회영역 분석으로 핵심 고객군과 서비스 요구를 구조화했습니다.

Role

서비스 기획, 데이터 수집·전처리·분석, 프론트엔드 구현, AI와 외부 API 연동, Supabase 데이터 설계와 시연 환경 구성을 담당했습니다.

BX·CX Analysis Flow

고객의 목소리를 서비스 기회로 바꾼 과정

시장과 고객 관점에서 문제를 정의한 뒤, 대규모 VOC를 정제하고 고객군·토픽·기회영역을 검증해 Mother Together의 서비스 기능으로 연결했습니다. 이미지를 클릭하면 크게 볼 수 있습니다.

분석 결과를 서비스 기능으로 연결했습니다.

신체 변화와 환경 부담에는 음성 기반 가전·조명 제어를, 정서적 불안과 관계 부담에는 배우자 알림과 상태 공유를, 지속적인 생활 관리에는 이벤트 로그와 기록 기능을 설계했습니다.

분석 결과를 실제 MVP로 구현

사용자의 음성 발화가 AI 의도 분석을 거쳐 ThinQ 공기청정기, Philips Hue 조명, 3D 홈 화면, Supabase 기록 저장으로 이어지도록 연결했습니다. 이미지를 클릭하면 크게 볼 수 있습니다.

Next.js Vercel Supabase OpenAI API LG ThinQ API FastAPI Python 3D Simulator Philips Hue Pandas Text Preprocessing Embedding PCA UMAP Ward Clustering Silhouette Score Topic Modeling Opportunity Analysis
핵심 포인트. 469,020건의 고객 데이터에서 문제와 기회영역을 도출하고, 그 결과를 음성 입력 → AI 판단 → 가전·조명 제어 → 3D 공간 반응 → 실행 로그 저장으로 이어지는 시연 가능한 MVP로 구현했습니다.

Problem

스탠바이미 Go처럼 고객이 새로운 사용 상황을 떠올리게 하는 제품 기회를 또 발견할 수 있을까? 또한 LGE.COM에서 고객이 탐색·구매 과정 중 어떤 지점에서 불편을 느끼는지 데이터 기반으로 개선할 수 있을까?

Built

커뮤니티와 리뷰 데이터를 수집·전처리하고, 고객이 실제로 표현한 상황, 불편, 니즈를 분류해 제품 기회와 UX 개선 포인트로 구조화했습니다.

Result

BX 프로젝트에서는 여행 중 의류 관리 니즈를 바탕으로 AI Styler Go 컨셉을 도출했고, CX 프로젝트에서는 고객이 제품 정보를 찾고 비교한 뒤 구매를 결정하는 과정에서 겪는 불편을 분석해 LGE.COM의 탐색 흐름, 정보 배치, 구매 전환 UX 개선 방향을 제안했습니다.

핵심 포인트. 단순히 데이터를 수집하는 데서 끝내지 않고, 실제 사용자의 언어를 기반으로 니즈와 페인포인트를 정리한 뒤 제품 컨셉과 UX 개선 방향으로 연결했습니다.
Project Focus

제가 보여드리고 싶은 강점

“특정 기술을 많이 썼다”보다, 문제를 이해하고 실제 동작하는 흐름으로 연결한 경험을 중심으로 작성했습니다.

01

문제 정의

기술을 먼저 정하지 않고, 사용자가 어떤 상황에서 불편을 겪는지부터 정의합니다.

02

데이터 기반 사고

사용자 발화, 리뷰, 커뮤니티 데이터를 분석해 문제를 구조화하는 경험을 쌓았습니다.

03

API 연동

AI 판단 결과를 외부 API와 연결해 실제 화면과 기기가 반응하도록 구현했습니다.

04

서비스 구현

분석 결과를 문서에서 끝내지 않고, 사용자가 체감할 수 있는 서비스 흐름으로 연결합니다.
